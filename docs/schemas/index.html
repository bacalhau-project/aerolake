üîç Pipeline Metadata Fields

Every record processed includes audit metadata:

  • pipeline_version - Version of the processing pipeline
  • pipeline_stage - Which stage processed this record (raw, schematized, validated, etc.)
  • processing_timestamp - ISO 8601 timestamp of when processed
  • git_sha - Git commit hash (40 characters) of pipeline code
  • node_id - Identifier of the processing node
  • transformation_hash - SHA256 hash of transformation logic for reproducibility

‚ö° Business Logic Validation Rules

The schema enforces physics-based constraints:

Usage Examples

Python Validation

import requests
import jsonschema

# Fetch schema from aerolake.org
schema_url = "https://aerolake.org/schemas/wind-turbine-v1.json"
schema = requests.get(schema_url).json()

# Your data
data = {
    "timestamp": "2024-01-01T12:00:00Z",
    "turbine_id": "WT-0001",
    "site_id": "SITE-TEX",
    "temperature": 25.5,
    "humidity": 65,
    "pressure": 1013.25,
    "wind_speed": 12.5,
    "power_output": 2500,
    "pipeline_metadata": {
        "pipeline_version": "1.0.0",
        "pipeline_stage": "validated",
        "processing_timestamp": "2024-01-01T12:00:05Z",
        "git_sha": "abc123def456...",
        "node_id": "node-001",
        "transformation_hash": "sha256hash..."
    }
}

# Validate
try:
    jsonschema.validate(data, schema)
    print("‚úÖ Data is valid!")
except jsonschema.ValidationError as e:
    print(f"‚ùå Validation failed: {e.message}")
